Publiceringsplattform - Render

Jag har valt att använda plattformen Render som publiceringsverktyg efter tips från en kurskamrat. Render finns att använda gratis där olika typer av tjänster erbjuds som bland annat statiska webbplatser och webbtjänster med mera. Det går också att arbeta i team men då kickar betalversionerna in. 19 dollar i månaden för små grupper, 29 dollar för större grupper och så vidare. Detta är första gången jag publicerar något via Render och jag blev positivt överraskad över hur lätt det var att komma igång samt koppla ett repository via Github. Render går nämligen att använda ihop med Git, därav blir projekten väldigt lättarbetade då arbetsprocessen blir i princip omedelbar och det tar bara någon minut innan ändringar syns på webbplatsen. En väldigt stor stor fördel i mina ögon.

Render stöder:

  • Node.js
  • Python
  • Ruby
  • Elixir
  • Go
  • Rust
  • PHP

Render kan också lägga upp Dockerfiler och det finns möjlighet att använda Laravel ihop med Docker för att lägga upp PHP-webbapplikationer. Jag har läst på flera webbsidor att Render används som gratis publiceringsplattform för Laravel nu istället för Heroku. Eftersom jag inte provat Render så mycket än, kan jag inte direkt peka ut någon som skulle kunna vara en nackdel, men från det jag läst så verkar det vara önskvärt med stöd för fler programspråk och att gratisversionen är lite långsam, vilket jag än inte upplevt, men mitt tålamod kanske är större. Än så länge tycker jag Render är en bra plattform med ett användarvänligt gränssnitt och med mycket dokumentation till hjälp.
